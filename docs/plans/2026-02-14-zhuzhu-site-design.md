# zhuzhu Site - 设计方案

**创建日期**: 2026-02-14  
**项目类型**: 情侣记录网站（纯静态前端）  
**架构**: 单页应用 (SPA)  
**恋爱起始日**: 2024年9月19日

---

## 1. 项目概述

### 1.1 核心功能
- 🏠 **首页**: 恋爱天数倒计时 + 精选照片轮播
- 💕 **纪念日**: 重要日期展示（周年、100天等）
- 📸 **相册墙**: 瀑布流照片展示（通过 Notion API 获取）
- ⏳ **时光轴**: 恋爱历程时间线
- 💌 **留言板**: 甜蜜留言互动

### 1.2 设计风格
- **主题**: 复古胶片风 (Retro Film)
- **感觉**: 温暖、怀旧、电影质感
- **动画**: 柔和舒缓（像在翻阅相册）
- **响应式**: 支持手机、平板、电脑

### 1.3 技术栈
- **框架**: Vue 3 + Composition API
- **构建**: Vite 5
- **样式**: Tailwind CSS 3
- **动画**: GSAP + @vueuse/motion
- **后端**: Notion API
- **部署**: Netlify (CI/CD 自动部署)

## 2. 视觉设计系统

### 2.1 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| **主色调** | `#D4A574` (复古金) | 强调色、按钮、高亮 |
| **背景色** | `#F5EDE4` (米白/暖白) | 主背景，像旧相纸 |
| **卡片色** | `#FFF8F0` (奶油白) | 卡片、内容区 |
| **文字主色** | `#3D3D3D` (深灰) | 正文 |
| **文字次色** | `#8B7355` (暖棕) | 次要文字、日期 |
| **边框色** | `#E8DDD0` (浅褐) | 分隔线、边框 |

### 2.2 字体搭配

- **标题**: Playfair Display（优雅衬线体，复古杂志感）
- **正文**: Inter（现代柔和无衬线，保证可读性）
- **手写**: Caveat（用于甜蜜短语、日期标签）

```css
@import url('https://fonts.googleapis.com/css2?family=Caveat:wght@400;700&family=Inter:wght@300;400;600&family=Playfair+Display:wght@400;600;700&display=swap');
```

### 2.3 质感效果

- 轻微颗粒感背景 (film grain overlay)
- 柔和阴影（像照片浮在页面上）
- 圆角卡片设计（8-16px，温暖友好）
- 玻璃拟态导航栏（backdrop-filter: blur）

## 4. 各页面详细设计

### 4.1 🏠 首页 (Hero)

**内容**:
- 大标题："我们在一起"
- 天数计数器（实时计算：2024.9.19 至今）
- 1-2张精选照片（轮播）
- 向下滚动提示

**动画**:
- 页面加载：文字从下方淡入 (duration: 1s)
- 天数数字：从0滚动到实际天数
- 背景照片：缓慢缩放 (scale 1.0 → 1.05, 20s循环)
- 滚动提示：轻柔上下浮动

**响应式**:
- 移动端：字体缩小30%，照片单张轮播，导航移至底部

---

### 4.2 💕 纪念日页面

**内容**:
- 大数字展示：在一起总天数（实时计算）
- 卡片列表：
  - 100天纪念日
  - 200天纪念日
  - 周年纪念日
  - 生日等

**布局**:
- 桌面端：3列网格
- 移动端：单列卡片堆叠

**动画**:
- 滚动进入：卡片依次从下方淡入 (stagger: 100ms)
- 悬停：卡片轻微上浮 + 阴影加深
- 倒计时数字：脉冲动画（接近重要日期时）

**数据**: Notion "纪念日" 数据库

---

### 4.3 📸 相册墙页面

**内容**: 照片瀑布流展示

**布局**:
- 桌面端：Pinterest风格瀑布流 (Masonry)，3-4列
- 平板端：2列
- 手机端：单列

**交互**:
- 点击照片：灯箱弹窗查看大图
- 支持手势：左右滑动切换照片
- 双指缩放查看细节

**动画**:
- 照片加载：从透明到可见 (opacity + scale)
- 瀑布流：新照片从底部滑入
- 灯箱：背景淡黑 + 照片从点击位置放大展开

**数据**: Notion "相册" 数据库（图片通过 API 获取）

---

### 4.4 ⏳ 时光轴页面

**内容**: 恋爱历程重要节点

**布局**:
- 桌面端：居中时间线，左右交替显示事件
- 移动端：单侧时间线（左侧线条，右侧内容）

**动画**:
- 滚动触发：节点从侧边滑入 (translateX)
- 时间线：随滚动从上往下绘制 (SVG stroke-dashoffset)
- 节点图标：轻微弹跳出现

**数据**: Notion "时光轴" 数据库

---

### 4.5 💌 留言板页面

**内容**: 留言互动

**布局**:
- 顶部：写留言输入框（可展开）
- 下方：留言卡片流（时间倒序）

**桌面端**: 2列网格展示留言  
**移动端**: 单列卡片

**动画**:
- 新留言：从顶部滑入，带弹性效果
- 发送按钮：点击时轻微缩放反馈
- 留言卡片：悬停时轻微放大

**数据**: Notion "留言" 数据库

## 5. 数据架构（Notion 后端）

### 5.1 数据库设计

#### 📅 数据库 1：纪念日 (Anniversaries)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 名称 | Title | 纪念日名称 |
| 日期 | Date | 具体日期 |
| 类型 | Select | 周年/100天/生日/自定义 |
| 描述 | Rich Text | 简短描述（可选） |
| 图标 | Files | 小图标（可选） |

#### 📸 数据库 2：相册 (Photos)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | Title | 照片标题 |
| 图片 | Files | 照片文件（Notion 托管） |
| 日期 | Date | 拍摄日期 |
| 标签 | Multi-select | 地点/活动标签 |
| 描述 | Rich Text | 照片故事 |

#### ⏳ 数据库 3：时光轴 (Timeline)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 事件 | Title | 事件标题 |
| 日期 | Date | 发生日期 |
| 描述 | Rich Text | 详细描述 |
| 图标 | Emoji/Icon | 事件类型图标 |
| 重要度 | Select | 高/中/低 |

#### 💌 数据库 4：留言 (Messages)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 内容 | Title | 留言内容 |
| 发送者 | Select | zhuzhu/对方 |
| 时间 | Created Time | 自动记录 |
| 心情 | Select | 开心/感动/想念等 |

### 5.2 数据流

```
Notion API → 前端缓存 (SWR) → UI渲染
   ↑                              ↓
   └──── 用户提交新数据 ←─────────┘
```

**更新策略**:
- 页面加载时从 Notion 获取最新数据
- 使用 SWR 进行本地缓存和自动重新验证
- 用户提交数据后直接写入 Notion
